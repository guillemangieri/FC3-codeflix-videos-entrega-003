Quarta entrega

 Neste desafio, você deve avançar no desenvolvimento da API implementando os endpoints (CRUD) completos para as entidades Categoria e Cast Member (Membros do Elenco). Utilize a implementação de Categoria vista em aula como referência arquitetural para replicar os padrões em Cast Member.

Tecnologias Obrigatórias

  • Linguagem: TypeScript

  • Testes: Jest

  • Validação: Class Validator

 

Requisitos e Comportamento

  1. Entidade Cast Member Você deve criar a entidade CastMember representando os atores e diretores do catálogo. Ela deve possuir os seguintes campos e regras:

 

  • name: Tipo string, obrigatório, com tamanho máximo de 255 caracteres.

  • type: Deve ser implementado como um Value Object ou Enum.

    • Valor 1: Representa Diretor.

    • Valor 2: Representa Ator.

  • created_at: Data de criação do registro.

  • Validações: Aplique todas as validações de domínio necessárias (campos obrigatórios, tipos, tamanhos) conforme demonstrado no módulo de Categoria.

  • Funcionalidades da API (CRUD) Implemente os endpoints para criar, atualizar, buscar e remover Cast Members. Para o endpoint de Listagem, deve ser possível:

    • Ordenar os resultados por name ou created_at.

    • Filtrar os resultados por name ou type.

  • Qualidade e Testes (QA)

    • Aplicar a Pirâmide de Testes completa (Unitários, Integração e E2E, conforme estrutura do curso).

    • Manter a cobertura de testes (Code Coverage) em no mínimo 80%.

    • Garantir que não existam erros de tipagem no projeto.
